Deep Dive: Technology Stack Justification

Core AI Engine
Google Gemini 3.5 Flash

How it works: Gemini ingests the image of the prescription directly and uses its native multimodal capabilities to output a structured JSON of medicines, dosages, and patient details.

Why we chose this: Legacy OCR models (like Tesseract) fail completely on Indian doctor handwriting. Gemini 3.5 Flash not only reads the handwriting with near-perfect accuracy across 6 regional languages, but it understands the context of medical abbreviations (e.g., "BD" = Twice Daily). Its sub-second latency is crucial for our real-time 3-second SLA.

Backend Architecture
FastAPI & Python

How it works: A high-performance asynchronous Python backend serves the API endpoints for the clinics and patient portals, managing the flow between the UI and the AI agents.

Why we chose this: We needed a lightweight, async framework that natively supports WebSockets (for future real-time alerts) and integrates perfectly with Google's Python SDKs. FastAPI's auto-generated Swagger documentation also makes future third-party integrations trivial.

Database & Context
MongoDB Atlas & MCP

How it works: All patient longitudinal records, including allergies and past prescriptions across different clinics, are stored as flexible JSON documents.

Why we chose this: Healthcare data is rarely uniform. Relational SQL databases break when dealing with varying lab report structures. MongoDB allows us to maintain a flexible schema. We use Model Context Protocol (MCP) to let our AI agents securely query the database in natural language.

Security & Orchestration
Google Cloud ADK & KMS

How it works: The Google Agent Development Kit orchestrates the 4-agent pipeline. Cloud KMS handles envelope encryption for all Personally Identifiable Information (PII).

Why we chose this: To deploy in Indian hospitals, we must comply with ABDM data privacy laws. KMS ensures data is encrypted at rest. The ADK allows us to strictly define boundaries for our Safety Agent, ensuring it relies on deterministic WHO EML data rather than hallucinating medical advice.
