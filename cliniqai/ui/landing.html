Tech Stack Deep Dive

Every technology was chosen to solve specific clinical challenges in rural India. No hype, just impact.

Core AI Engine

Google Gemini 3.5 Flash via Vertex AI

  • How it works: Gemini ingests the raw image and uses native multimodal capabilities to output a structured JSON schema.
  • Why we chose it: Legacy OCR models (Tesseract) fail completely on Indian doctor handwriting. Gemini 3.5 Flash not only reads 6 regional languages but understands medical context ("BD" = Twice Daily). Sub-second latency is crucial for our 3-second SLA.
Orchestration

Google Agent Dev Kit (ADK)

  • How it works: Orchestrates the Supervisor and 4 specialized agents. Maps our Python functions as `FunctionTool` and connects to MongoDB via `McpToolset`.
  • Why we chose it: Single-LLM prompts are dangerous for healthcare. ADK allows us to build a deterministic system where agents run in parallel and the SafetyAgent provides a hard, algorithmic validation gate.
Database & Context

MongoDB Atlas & MCP

  • How it works: Patient histories are stored as JSON documents. The AI queries the DB in natural language using the Model Context Protocol (MCP).
  • Why we chose it: Medical data (lab reports, prescriptions) varies wildly in structure. Relational SQL breaks. MongoDB's flexible schema handles this gracefully, and MCP integrates flawlessly with ADK.
Security & Orchestration

FastAPI, Cloud Run & KMS

  • How it works: High-performance async Python backend deployed on serverless Cloud Run. Cloud KMS handles envelope encryption.
  • Why we chose it: FastAPI's async nature perfectly supports `asyncio.gather` for parallel agent execution. Cloud KMS ensures ABDM-compliant encryption for patient PII, meaning data is safe at rest.

True Data Sovereignty & Security

Patients OWN their data, not hospitals. Our architecture enforces strict sovereignty:

  • 📱
    OTP-Based Access Control:

    Mobile numbers act as the universal ID. A 5-minute OTP grants access. Patients can revoke clinic access instantly. No Aadhaar required for MVP.

  • ⛓️
    SHA-256 Hashing:

    Every database write creates an immutable hash-chain audit log. Tampering is mathematically impossible without breaking the chain.

  • 🔒
    Cloud KMS Encryption:

    All PII is envelope-encrypted at rest. Even database administrators cannot read raw patient data.
